# Second Brain 自動流程

這個 vault 已加入本地外掛 `Second Brain Pipeline`。

## 用法

1. 在 Obsidian 重新載入外掛。
2. 到 `Settings -> Community plugins` 確認 `Second Brain Pipeline` 已啟用。
3. 到外掛設定填入 Gemini API key，或先在系統環境變數設定 `GEMINI_API_KEY`。
4. 用 command palette 執行以下命令：
   - `Second Brain Pipeline: Run full pipeline`
   - `Second Brain Pipeline: Run Phase 1 ingest`
   - `Second Brain Pipeline: Run Phase 2 compile`
   - `Second Brain Pipeline: Run Phase 3 query`
   - `Second Brain Pipeline: Run queued queries`
   - `Second Brain Pipeline: Run Phase 4 lint`

## 自動流程約定

- `raw/`：原始文件輸入區
- `staging/`：Phase 1 會把可讀文字檔複製到這裡
- `wiki/articles/`：來源文章整理結果
- `wiki/概念/`：概念筆記
- `wiki/queries/`：查詢歸檔
- `wiki/lint/`：lint 報告
- `system/queue/_queue.md`：Phase 3 批次查詢佇列
- 預設所有人類可讀輸出皆使用繁體中文，必要時才保留標準編號、英文縮寫或專有名詞原文

## PDF 支援

- PDF 不會被複製到 `staging/`，而是由 Phase 2 直接從 `raw/` 或 `staging/` 讀取後送進 Gemini。
- 目前採用 Gemini 官方文件理解能力的 inline PDF 方式，較適合小到中型 PDF。
- 若 PDF 超過外掛設定中的 `Max inline PDF bytes`，該檔會被略過並在 console 留下原因。

## 忙碌重試

- 若 Gemini 回 `429` 或 `5xx`，外掛會依設定自動重試。
- 可在外掛設定調整 `Retry count` 與 `Retry base delay ms`。

## 增量模式

- Phase 2 現在會為每個來源檔建立內容指紋快取。
- 若來源檔內容未變更，重新執行 `Run full pipeline` 時會跳過該檔，不再重複送 Gemini。
- 若來源檔內容已變更，外掛會重新編譯該檔，並在成功後清掉同一來源先前產生且已過時的 article 與 concept 輸出。

## 佇列查詢

若要讓 full pipeline 也自動跑 Phase 3，可編輯 `system/queue/_queue.md` 並使用：

```md
- [ ] 什麼是 LLM 編譯器？
- [ ] 這個系統為什麼不使用向量資料庫？
```

外掛執行 `Run queued queries` 或 `Run full pipeline` 後，會把已完成項目標記為完成並附上答案筆記連結。
