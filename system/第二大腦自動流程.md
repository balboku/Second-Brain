# 🚀 Second Brain 自動流程指南

這個 Vault 已深度整合本地外掛 `Second Brain Pipeline`，將其變成一個由 LLM 驅動的文章編譯與維修系統。

## 🛠️ 初始化設定

1. **啟用外掛**：到 `Settings -> Community plugins` 確認 `Second Brain Pipeline` 已啟用。
2. **API 金鑰**：在外掛設定填入您的 **Gemini API Key**。
3. **模型配置**：
   - **Gemini Model**：用於 Phase 2 文章重寫與 Phase 4 分析。
   - **Gemini Embedding Model**：用於 Phase 3 的向量搜尋（預設為 `gemini-embedding-2-preview`）。建議一旦確定模型後不要輕易更改，以免向量不一致。
4. **自動修復開關**：勾選 `Enable Auto-Fix` 以便在 Phase 4 自動執行連結標準化與字典對位。

## 📋 指令清單 (Command Palette)

- `Run full pipeline`：自動執行 P1 -> P2 -> Queued Queries -> P4。
- `Run Phase 1 ingest`：將 `raw/` 讀取並標準化至 `staging/`。
- `Run Phase 2 compile`：利用 AI 重構文章與提煉概念。
- `Run Phase 4 lint`：執行用語字典對位與自主修復迴圈。系統會自動補齊空殼存根與縫合孤立頁面，持續迭代直至 0 Issue。

## 🗂️ 重要目錄約定

- `wiki/概念/`：存放所有原子化的知識點筆記。
- `wiki/articles/`：存放來源文件的結構化摘要與目錄。
- `system/glossary.json`：**用語對位標竿** (已排除 Git 追蹤)。所有同義詞的標準化規則皆儲存於此，由 AI 自動更新。
- `wiki/lint/`：存放自動化修復與 AI 分析的最新報告。

## 🧠 進階特性

### 自動詞彙擴充 (Glossary)
系統會自動學習您的同義詞。當 AI 在 Phase 2 發現新術語與現有概念相似時，會自動在 `glossary.json` 建立映射規則。在下一次執行 Phase 4 時，全維基中的該術語就會自動被替換為標準詞。

### 自主修復錄迴圈 (Self-Healing Loop)
Phase 4 指令不再只是產出報告：
- **自動對位**：依據 `glossary.json` 修復全域斷裂連結。
- **孤立縫合**：自動在孤立的概念頁面中插入反向連結。
- **存根自動擴充**：偵測到內容貧乏的存根頁面時，系統會自動追溯來源 PDF 並發動同步 Phase 2 重編譯。
- **持續迭代**：修復後若仍有殘留 issue，系統會自動發動下一輪 Lint 與修復，直到完美狀態。
