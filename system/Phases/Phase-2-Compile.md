# Phase 2: 編譯 (Compile)

LLM 作為編譯器，逐步將 raw/ 中的原始文件轉換為結構化維基。

## 核心功能

### 索引與摘要
- 自動維護索引文件
- 包含所有文件的簡要摘要
- 作為查詢的入口點

### 概念文章
- 目標：~100 篇概念文章，~40 萬字
- 按主題組織
- 包含反向連結與交叉引用

### 衍生產出
- Marp 簡報
- matplotlib 圖表
- 查詢答案歸檔

### 連結系統
- 自動生成概念間的連結圖
- 找出新文章的連接候選
- 建立主題叢集

## 工作流程

```
raw/ --> LLM 讀取 --> 結構化維基 --> wiki/
                    (編譯器)
```

## 操作步驟

1. **準備 staging/**
   - 從 raw/ 取文件放入 staging/

2. **執行編譯**
   - 提示 LLM 讀取 staging/ 中的文件
   - 要求生成結構化摘要與概念文章
   - 指定輸出至對應目錄

3. **更新索引**
   - 自動更新 `wiki/全域索引.md`
   - 確保反向連結正確

## 產出目錄

| 目錄 | 內容 |
|------|------|
| `wiki/概念/` | 主題概念文章 |
| `wiki/articles/` | 原始文章整理結果 |
| `wiki/全域索引.md` | 全域索引 |
| `wiki/slides/` | Marp 簡報 |
| `wiki/charts/` | 圖表 |
