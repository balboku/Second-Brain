# Phase 2: 編譯 (Compile)

LLM 作為編譯器，將原始文件轉換為標準化的結構化維基，並同時維護詞彙一致性。

## 核心功能

### 概念提煉 (Structural Extraction)
- 將來源文件重構為具備 YAML Metadata 的概念筆記。
- **深度數據擷取**：自動提煉樣本數 (n)、研究設計與統計顯著性 (p-values)。
- **分析型輸出**：針對對照研究，要求採用 Markdown 表格進行多維度比較。
- 自動提取關鍵字與標籤，並優先對位全域字典。
- 建立雙向連結與相關概念引用。

### 詞彙字典自動擴充 (Glossary Expansion)
- **術語一致性**：系統會依據 `system/glossary.json` 對位標準詞彙，確保 Wiki Link 指向正確的概念。
- **標籤規一化**：系統自動處理標籤格式（如處理空格為連字號 `-`），維護全域標籤的一致性。
- **自動建議同義詞**：系統在分析文獻時若發現潛在的同義詞，會主動記錄並提供合併建議。
- **字典自動更新**：確認後的詞彙映射將自動寫入 `system/glossary.json`，在下一次掃描時全域生效。

### 索引與摘要 (Indexing & References)
- 自動維護 `wiki/articles/` 摘要文章，包含研究核心結論。
- 確保所有術語遵循「中文名 (英文名)」標題規範，避免編碼與歧義問題。
- 作為全域索引的資料來源。

## 工作流程

```
raw/ --> LLM 編譯器 --> 結構化維基 (wiki/)
           ↑
    讀取 glossary.json
           ↓
    建議新對應關係 --> 更新 glossary.json
```

## 操作步驟

1. **準備來源**
   - 將文字或 PDF 放入 `raw/` 或 `staging/`。

2. **執行編譯**
   - 執行 `Run Phase 2 compile` 或 `Run full pipeline`。
   - 系統會自動執行字典注入與自動建議解析。

3. **檢查字典**
   - 可隨時開啟 `system/glossary.json` 查看 AI 自動整理的詞彙映射。

## 產出目錄

| 目錄 | 內容 |
|------|------|
| `wiki/概念/` | 主題概念筆記 |
| `wiki/articles/` | 來源文章的結構化摘要 |
| `wiki/全域索引.md` | 全域入口點 |
