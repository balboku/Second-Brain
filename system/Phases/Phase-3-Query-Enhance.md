# Phase 3: 查詢與增強 (Query & Enhance)

使用維基系統進行研究與知識探索。

## 工具

### Obsidian IDE
- 維基的前端介面
- 瀏覽文章與可視化
- 管理反向連結

### Q&A Agent
- 處理複雜研究問題
- 橫跨多篇文章查詢
- 答案以 Markdown、簡報或圖表呈現

### 搜尋引擎
- 輕量級全文搜尋
- Web UI 或 CLI 介面
- 可供 LLM 查詢使用

## 工作流程

```
查詢 --> Q&A Agent --> 維基檢索 --> 答案 --> 歸檔至 wiki/queries/
                                   --> 衍生產出（簡報/圖表）
```

## 操作步驟

1. **提出問題**
   - 使用 CLI 或 Web UI 提交查詢
   - 或直接透過 Obsidian 瀏覽
   - 若要批次執行，可在 `system/queue/_queue.md` 維護待處理問題

2. **獲取答案**
   - Q&A Agent 檢索相關概念
   - 生成結構化答案
   - 必要時生成簡報或圖表

3. **歸檔答案**
   - 將問答結果存入 `wiki/queries/`
   - 視需要生成 `wiki/slides/` 或 `wiki/charts/`
   - 確保答案可被未來查詢引用

## 查詢歸檔

所有輸出都會存入 `wiki/queries/`，形成累積知識庫。

## 未來方向

系統輸出的合成數據可用於微調 LLM，使其「內化」這些知識。
