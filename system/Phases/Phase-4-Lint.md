# Phase 4: 檢查與修復 (Lint & Fix)

透過「確定性腳本」與「AI 分析」雙重機制，自動維護維基系統的一致性並消除重複與錯誤。

## 核心功能

### 確定性自動修復 (Auto-Fix)
此階段在不消耗 API Token 的情況下執行：
- **用語標準化對位**：根據 `system/glossary.json`，將所有非標準詞彙連結（如 `[[Breast Surgery]]`）自動替換為標準詞彙。
- **標籤自動修正**：系統會自動辨識並替換標籤（如 `#Breast-Surgery` 或 `#Breast_Surgery`），將其統一為標準詞彙的連字號標籤格式。
- **斷裂連結修復**：自動校正大小寫錯誤、路徑誤差，並與現存筆記匹配。
- **存根頁面建立 (Stub Creation)**：對於完全找不到對應的概念連結，自動建立具備 YAML 結構的存根頁面，消除斷開的連結。

### 詞彙字典自動擴充 (Glossary)
- **自動建議**：LLM 在編譯過程中如果發現文中使用了同義詞（如「AI」與「人工智慧」），會自動回傳建議。
- **標準化生成**：系統會要求 LLM 嚴格遵守字典，生成 [[雙向連結]]；若生成 #標籤，系統會自動將空白轉換為連字號 `-`（例如 `#Breast-Surgery`）。
- **字典合併**：外掛會自動將建議詞彙寫入 `system/glossary.json`。

## 工作流程

```
wiki/*.md --> [用語對位腳本] --> [斷裂連結修復腳本] --> [AI 掃描分析] --> 報告與建議
                  ↑                    ↑                   │
           讀取 glossary        建立存根頁面          建議新詞彙對應
```

## 操作步驟

1. **執行 LINT**
   - 執行 `Run Phase 4 lint` 命令。
   - 外掛會先顯示「正在執行自動連結修復與用語對位...」通知。

2. **檢閱報告**
   - 查看 `wiki/lint/report-YYYYMMDD.md`。
   - 報告中的 **🔧 Auto-Fix 自動修復紀錄** 會詳細列出所有被修正的檔案路徑。

3. **確認字典更新**
   - 若 AI 發現新同義詞，會自動合併回 `system/glossary.json`，在下一次 Lint 時就會自動執行全域替換。
